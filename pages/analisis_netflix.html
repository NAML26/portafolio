  3. Cardinalidad: cuántos valores distintos tiene cada columna categórica.
  4. Consistencia: si los tipos de datos coinciden con el significado de la columna.

4. Limpieza de datos¶

Vamos a limpiar paso a paso y documentar cada decisión.

Decisiones de limpieza¶

  1. duplicados: eliminamos filas repetidas porque no aportan información nueva.
  2. date_added: convertimos a tipo datetime para poder analizar tendencias temporales.
  3. country: muchos nulos; los reemplazamos por Unknown para no perder registros.
  4. rating: algunos nulos; los reemplazamos por Unrated para mantener consistencia.
  5. duration: homogenizamos espacios y formato para evitar inconsistencias.
  6. Exportación: guardamos el dataset limpio como netflix_clean.csv para usarlo en la app de Streamlit.

Idea clave: no existe una sola forma "correcta" de limpiar. Cada decisión cambia el análisis. Por eso documentamos el criterio.

5. Creación de variables nuevas¶

Muchas veces la información útil no viene lista en una sola columna. Hay que construirla.

Decisiones de creación¶

  1. year_added: extraemos el año de la columna date_added para analizar tendencias de incorporación a Netflix.
  2. month_added: extraemos el mes de date_added para ver estacionalidad en lanzamientos.
  3. has_director: variable binaria (1 si hay director registrado, 0 si está vacío).
  4. has_cast: variable binaria (1 si hay elenco registrado, 0 si no).
  5. main_genre: tomamos el primer género de la columna listed_in para simplificar análisis.
  6. duration_int y duration_type: separamos la columna duration en número (minutos o temporadas) y tipo.
  7. rating_group: agrupamos clasificaciones (PG, TV-MA, etc.) en categorías más amplias como Family o Adult.
  8. is_recent: indicador de si el título fue lanzado en los últimos 5 años (1 = reciente, 0 = antiguo).
  9. country_top: país con más películas de cierto rating, útil para ver patrones regionales.

Preguntas que responden estas variables¶

  • year_added → ¿En qué años Netflix añadió más contenido?
  • month_added → ¿Hay meses con más estrenos?
  • has_director → ¿Qué porcentaje de títulos tiene director registrado?
  • main_genre → ¿Cuál es el género principal más frecuente?
  • rating_group → ¿Qué tipo de contenido domina: familiar o adulto?
  • is_recent → ¿Qué proporción de títulos son recientes?
  • country_top → ¿Qué países producen más películas con clasificación alta?

7. Regresión logística de contenido reciente¶

Ajustamos un modelo logístico para interpretar qué variables se asocian con que un título sea reciente (is_recent). Antes de ajustar, corregimos dos problemas que aparecían en la versión original:

  • Valores faltantes en duration_int (la limpieza no rellenó los nulos de duration): se eliminan esas filas antes de ajustar, para que el modelo de interpretación y el de evaluación usen exactamente las mismas filas.
  • Separación cuasi-perfecta en main_genre: géneros poco frecuentes, o géneros donde is_recent es siempre 0 o siempre 1 (como Classic Movies), se agrupan en "Otros" para evitar coeficientes y errores estándar disparados (y el ConvergenceWarning).

import statsmodels.formula.api as smf
from sklearn.model_selection import train_test_split
from sklearn.linear_model import LogisticRegression
from sklearn.metrics import accuracy_score, confusion_matrix, classification_report, roc_auc_score


def ajustar_modelo_logit(df, min_freq_genero=50):
    """Ajusta un modelo de regresión logística sobre todo el dataset,
    para interpretar el efecto de cada variable.

    Los géneros con menos de `min_freq_genero` observaciones, o con `is_recent`
    constante, se agrupan en 'Otros' para evitar separación cuasi-perfecta.

    Devuelve el DataFrame reducido a las columnas usadas por el modelo,
    para reutilizarlo en la etapa de evaluación.
    """
    print("--- Regresión logística de contenido reciente ---")

    columnas_modelo = [
        "is_recent", "type", "main_genre", "rating_group",
        "duration_int", "has_director", "has_cast",
    ]
    modelo_df = df[columnas_modelo].copy()

    # 'duration' tiene valores faltantes que no se rellenaron en la limpieza,
    # así que 'duration_int' puede traer NaN. Los quitamos aquí para que el
    # modelo de interpretación y el de evaluación usen las mismas filas.
    filas_antes = len(modelo_df)
    modelo_df = modelo_df.dropna()
    filas_eliminadas = filas_antes - len(modelo_df)
    if filas_eliminadas:
        print(f"Filas eliminadas por valores faltantes: {filas_eliminadas}")

    # Agrupar géneros problemáticos en 'Otros'. Dos motivos causan separación
    # cuasi-perfecta y hay que cubrir ambos:
    #   (a) baja frecuencia: pocas observaciones, fácil que salgan todas del mismo lado
    #   (b) pureza: aunque tenga muchas observaciones, is_recent es constante
    #       dentro de esa categoría (ej. 'Classic Movies' es siempre antiguo)
    frecuencia_genero = modelo_df["main_genre"].value_counts()
    generos_poco_frecuentes = set(frecuencia_genero[frecuencia_genero < min_freq_genero].index)

    pureza_genero = modelo_df.groupby("main_genre")["is_recent"].nunique()
    generos_puros = set(pureza_genero[pureza_genero < 2].index)

    generos_a_agrupar = generos_poco_frecuentes | generos_puros
    modelo_df["main_genre"] = modelo_df["main_genre"].where(
        ~modelo_df["main_genre"].isin(generos_a_agrupar), "Otros"
    )
    if generos_a_agrupar:
        print(
            f"Géneros agrupados en 'Otros': {len(generos_a_agrupar)} "
            f"({len(generos_poco_frecuentes)} por baja frecuencia, "
            f"{len(generos_puros - generos_poco_frecuentes)} por ser puros/constantes)"
        )

    formula = (
        "is_recent ~ C(type) + C(main_genre) + C(rating_group) "
        "+ duration_int + has_director + has_cast"
    )
    modelo = smf.logit(formula, data=modelo_df).fit(disp=0)

    display(modelo.summary())
    print("\nPseudo R-cuadrado:", round(modelo.prsquared, 4))

    return modelo_df


def evaluar_modelo(modelo_df, test_size=0.2, random_state=42):
    """Entrena un modelo logístico sobre un split train/test y evalúa su desempeño
    fuera de muestra (accuracy, AUC-ROC, matriz de confusión, reporte de clasificación).

    Usa scikit-learn (no statsmodels) para poder aplicar class_weight='balanced':
    'is_recent' solo es 1 en ~7% de las filas, así que sin ponderar por clase el
    modelo aprende a predecir siempre 0 (accuracy alta pero recall de 'Reciente' = 0).
    """
    print("--- Evaluación con train/test split ---")

    X = pd.get_dummies(modelo_df.drop(columns=["is_recent"]), drop_first=True).astype(float)
    y = modelo_df["is_recent"]

    X_train, X_test, y_train, y_test = train_test_split(
        X, y, test_size=test_size, random_state=random_state, stratify=y
    )

    # class_weight='balanced' pondera cada clase por el inverso de su frecuencia,
    # para que la clase minoritaria ('Reciente') pese tanto como la mayoritaria.
    modelo_train = LogisticRegression(class_weight="balanced", max_iter=1000)
    modelo_train.fit(X_train, y_train)

    probs = modelo_train.predict_proba(X_test)[:, 1]
    preds = modelo_train.predict(X_test)

    print("Exactitud en prueba:", round(accuracy_score(y_test, preds), 4))
    print("AUC-ROC en prueba:", round(roc_auc_score(y_test, probs), 4))

    print("\nMatriz de confusión:")
    print(pd.DataFrame(
        confusion_matrix(y_test, preds),
        index=["Real: Antiguo", "Real: Reciente"],
        columns=["Pred: Antiguo", "Pred: Reciente"],
    ))

    print("\nReporte de clasificación:")
    print(classification_report(y_test, preds, target_names=["Antiguo", "Reciente"]))

    return modelo_train
